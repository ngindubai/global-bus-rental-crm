0 · Local run (do this first) Required

Free · 5 minutes · needs only Node 18+ — no Docker

PostgreSQL runs as a plain Node process (embedded-postgres) — nothing to install, no admin rights, no Docker. Data is stored in .pgdata/.

  1. Open a terminal in the project folder Bus-Rental-CRM.
  2. Install dependencies: npm install
  3. Start the database: npm run db:up — leave this terminal running (Postgres on port 5433).
  4. In a second terminal: create tables with npm run db:push
  5. Load sample data: npm run db:seed
  6. Start the app: npm run dev → open http://localhost:3200
  7. Log in with admin@globalbusrental.com / admin123 and change the password under Users.

1 · Production database Required to deploy

Free tier → $19–69/mo · ~10 minutes
  1. Go to neon.tech (recommended) or supabase.com and create a free account.
  2. Click New Project → name it gbr-crm → choose the region closest to your customers (e.g. Frankfurt/Bahrain for UAE).
  3. Copy the connection string (starts with postgresql://). On Neon, use the “Pooled connection” string.
  4. Paste it into .env as DATABASE_URL="…".
  5. Run npm run db:push then npm run db:seed against the new database.

2 · App hosting Required to go live

$7–25/mo · ~15 minutes
  1. Push this folder to a new GitHub repo (already a git repo — just git remote add origin … and git push).
  2. On Render → New → Web Service → connect the repo.
  3. Build command: npm install && npm run db:generate && npm run build. Start command: npm start.
  4. Add all the variables from your .env under Environment (especially DATABASE_URL and a strong JWT_SECRET).
  5. Generate a strong secret: run node -e "console.log(require('crypto').randomBytes(48).toString('hex'))" and paste as JWT_SECRET.
  6. Deploy. After first deploy, run npm run db:push once (Render Shell) to create tables.

3 · Vonage — inbound calls Optional

~$1/number + per-minute · ~20 minutes · powers Module 17
  1. Create an account at dashboard.vonage.com.
  2. Copy your API key and API secret from the dashboard home → into .env as VONAGE_API_KEY / VONAGE_API_SECRET.
  3. Buy a virtual number: Numbers → Buy numbers (in your customers’ country).
  4. Create a Voice Application (Applications → Create) and note the Application ID → VONAGE_APPLICATION_ID.
  5. Set the application’s Event URL to https://YOUR-DOMAIN/api/integrations/vonage/events (POST).
  6. Copy the Signature secret (Settings) → VONAGE_SIGNATURE_SECRET for webhook verification.
  7. Link the purchased number to the application. Test by calling it — a Call Log + lead match should appear under Calls.

4 · Stripe — customer payment links Optional

2.9% + $0.30/txn · ~15 minutes · powers Module 13
  1. Create/sign in at dashboard.stripe.com and complete business verification.
  2. Developers → API keys → reveal the Secret key (sk_live_… or sk_test_…) → STRIPE_SECRET_KEY.
  3. Developers → Webhooks → Add endpoint: URL https://YOUR-DOMAIN/api/payments/stripe-webhook (handler can be added in Phase 1.1); copy the signing secret → STRIPE_WEBHOOK_SECRET.
  4. Save. In a booking, click Stripe link — it now creates a real payment link for the outstanding balance.

5 · Live exchange rates Optional

Free tier (1,500/mo) · ~5 minutes · powers Module 14
  1. Sign up at exchangerate-api.com (free).
  2. Copy your API key → .env as FX_API_KEY (keep FX_PROVIDER="exchangerate-api").
  3. Rates are now fetched + cached automatically when quotes are built. You can still set manual overrides under Settings → Currency.

6 · AI assistant (Claude) Optional

~$20–150/mo usage · ~5 minutes · powers Module 22
  1. Go to console.anthropic.com → API Keys → Create key.
  2. Add billing (Plans & Billing) and a monthly spend limit so costs stay predictable.
  3. Paste the key into .env as ANTHROPIC_API_KEY. Model is preset to claude-opus-4-8 (change AI_MODEL to claude-haiku-4-5-20251001 for cheaper drafts).
  4. The AI buttons on each lead now produce live drafts instead of stub text. All AI output stays editable before use.
